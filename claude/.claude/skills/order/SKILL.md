# /order スキル — 物品購入依頼書の自動作成

見積書 PDF を読み取り、`依頼書作成.xlsx` の物品一覧シートへ品目を自動追記し、依頼書 PDF も自動生成する。

## 起動方法

```
/order [見積書PDFパス]
```

パスを省略した場合は `~/Downloads/` 内の PDF ファイルを検索し、**最終更新日時が最も新しい1件**を見積書として自動的に使う。
- 該当する PDF が1件もない場合はエラーとして終了し、パス指定を促す
- 複数ある場合はユーザーに確認せず最新の1件を選び、選んだファイル名を Step 1 の出力冒頭で明示する（内容が見積書と異なる場合は Step 3 のプレビューで気づける）

## ツールと定数

- xlsx: `~/Documents/budget/2026/依頼書作成.xlsx`
- スクリプト: `~/work/projects/admin/scripts/append_items.py`
- CLAUDE.md: `~/work/projects/admin/CLAUDE.md`（列定義・master 値の詳細）
- 見積書アーカイブ先: `~/Documents/budget/{年度}/提出書類/`（年度は依頼書xlsxと同じ年度フォルダ）

master テーブルの選択肢（2026年度）:
- **費目**: 教員研究費 / 重点研究費 / 共同研究費 / 受託研究費 / 教育経費 / 森林環境整備促進事業
- **節**: 需用費 / 役務費 / 使用料及び賃借料 / 負担金 / 備品購入費
- **単位の例**: 個 / 枚 / 本 / 式 / セット / 双 / 着 / 機 / 冊 / 人

## 処理フロー

### Step 1 — 入力PDFを特定し読み取る

`args` にパスが渡されていればそのファイルを、省略時は「起動方法」の自動検出ルールで特定したファイルを Read ツールで読み込む。
まずどのファイルを使うか（パス・更新日時）を1行で表示してから、以下を構造化抽出する:

| 抽出項目 | 見積書での読み方 |
|---------|----------------|
| 業者名 | 会社名（見積元。発行元と問い合わせ先が異なる場合は発行元を採用し、「〜経由」等の付記はしない） |
| 内税フラグ | 見積書の**消費税の計算方法**で判断。品目ごとに消費税を計算してから合計 → 1（内税） / 税抜小計に対して一括で消費税を計算 → 0（外税）。**確定はStep 3の検算で行う**（ここでは仮判定でよい） |
| 品名 | 品目の一般名を短く（下記ルール参照） |
| 規格・品質 | 型番・仕様・サイズ・色等（なければ空文字） |
| 数量 | 数量列 |
| 単位 | 単位列（なければ「個」） |
| 単価 | 見積書に印字されている金額をそのまま転記する（税込⇔税抜の換算計算はしない） |

**品名の転記ルール**:
- **一般名を短く**が原則。ブランド名・入数・「アスクル限定」等の販売チャネル情報は品名に含めない。
- 型番（セ-Y31Y 等）は規格・品質列へ移す。色・サイズは品目の識別に必要な場合のみ品名に残す。
- 入数・容量（「1袋（5冊入）」「1個（70枚入）」等）は単位か規格・品質で表現し、品名には不要。
- 例: 「花王 クイックルワイパー 取り替えドライシート 1パック（40枚入）」→ 品名「クイックルワイパー替えシート」、規格「40枚入」
- 例: 「測量野帳 レベルブック〈ブライトカラー〉黄 耐水 PP表紙 セ-Y31Y 1袋（5冊入）コクヨ」→ 品名「測量野帳レベルブック 黄（5冊入）」、規格「セ-Y31Y コクヨ」
- 例: 「OAクリーナー 除菌99% ウイルス除去 BOXタイプ ウェットティッシュ 厚手 1セット（60枚×5個）アスクル限定」→ 品名「除菌OAクリーナー ウェットティッシュ（60枚×5）」、規格「」

**注意**:
- 単価は見積書に書かれている数値をそのまま使う。内税（品目ごとに課税）の見積書なら品目ごとの税込単価が、外税（合計に一括課税）の見積書なら税抜単価がそのまま「単価」になる。依頼書側の消費税計算はこの前提で組まれているため、ここで独自に×1.1や÷1.1をしない。
- 品名が 2 段書き（英語 + 日本語）の場合は両方を含めた方を品名に、もう一方を規格・品質に入れるか、適宜判断。
- 見積番号・有効期限などは無視してよい。

### Step 2 — 不足項目を確認する

**見積書にない項目**（注文ごとに判断が必要）を AskUserQuestion で確認する。
以下をまとめて 1 回の質問で聞く（2–4 問）:

1. **費目**（様式を決める）: master の選択肢を提示
2. **節**: master の選択肢を提示
3. **使用目的**: 自由記述（例: 実験 / データ解析 / 授業 / フィールド調査）
4. **納入希望日**: YYYY-MM-DD 形式（目安: 当月末 or 翌月末）
5. **事業名**（任意）: 必要なければスキップ

**起案日**: 既定 = 今日の日付（`currentDate` から取得）。特に指定がなければ確認不要。

### Step 3 — プレビューと承認

金額が絡むため、**必ず確認を取る**。
以下の表を表示して AskUserQuestion で承認を求める:

```
## 追記内容プレビュー

- 注文先: {業者名}
- 費目: {費目}（{節}）
- 使用目的: {使用目的}
- 起案日: {起案日}
- 納入希望日: {納入希望日}
- 内税: {「内税○」 or 「外税○」}

| # | 品名 | 規格・品質 | 数量 | 単位 | 単価 | 金額 |
|---|------|-----------|------|------|-----------|-----------|
| 1 | ... | ... | ... | ... | ...円 | ...円 |
...
| | 小計 | | | | | {小計}円 |
| | 消費税(10%) | | | | | {税額}円（内税の場合は0円） |
| | **合計** | | | | | **{合計}円** |
```

内税フラグと単価は、見積書に記載されている**税抜金額・税込金額の両方**を使って以下の検算で確定させる（**端数の差は一切許容しない**）:

1. 内税候補: 各品目の**税込単価**（見積書記載、無ければ `round(税抜単価 × 1.1)`）× 数量 を全品目で合計
2. 外税候補: 各品目の**税抜単価**（見積書記載）× 数量 を全品目で合計した小計に対して `round(小計 × 1.1)` を加算

どちらか一方が見積書の総合計と**完全に一致する**方を採用し、内税フラグを確定させる。単価には一致した側の金額（内税なら税込単価、外税なら税抜単価）を、見積書に印字された通りそのまま記録する。両方とも一致しない場合は、単価・数量の抽出誤りを疑い Step 1 に戻って修正する（安易に近い方を選ばない）。

### Step 4 — xlsx へ追記

承認が得られたら、以下の JSON を生成して `append_items.py` を呼び出す。

JSON 構造:
```json
{
  "order": {
    "費目": "...",
    "事業名": "...",
    "節": "...",
    "使用目的": "...",
    "起案日": "YYYY-MM-DD",
    "納入希望日": "YYYY-MM-DD",
    "業者名": "...",
    "内税": 0
  },
  "items": [
    {"品名": "...", "規格・品質": "...", "数量": N, "単位": "個", "単価": NNNN},
    ...
  ]
}
```

実行コマンド（`--json` にシングルクォートで JSON 文字列を渡す）:
```bash
python3 ~/work/projects/admin/scripts/append_items.py \
    ~/Documents/budget/2026/依頼書作成.xlsx \
    --json '<JSON文字列>'
```

stdout に新しい OrderID が出力される。stderr にバックアップパスと保存完了メッセージが出る。

**[WARN] が出た場合**: 業者名が master 未登録。`依頼書作成.xlsx` を開いた際に
master シートへ追記するよう案内する。

xlsx への追記が成功したら、続けて**入力に使った見積書PDFそのもの**を以下の名前でアーカイブにコピーする（Step 1 で読み込んだファイルをコピーするだけで、新規生成しない）:

```bash
cp "<Step1で読み込んだPDFパス>" \
   ~/Documents/budget/{年度}/提出書類/{起案日 YYYYMMDD}_{業者略称}_見積書.pdf
```

- `{年度}`: xlsx と同じ年度フォルダ
- `{業者略称}`: Step 5 で使う略称生成ロジックと同じもの（例:「株式会社横山事務器」→「横山事務器」）
- コピー元PDFは削除しない（Downloads から自動検出した場合も残したままにする）

### Step 5 — 依頼書 PDF を自動生成

xlsx 追記・見積書アーカイブ完了後、AppleScript で Excel を操作し依頼書シートを PDF に書き出す。

**出力パス**: `~/Documents/budget/{年度}/提出書類/{起案日YYYYMMDD}_{業者略称}_依頼書.pdf`

```bash
XLSX_PATH="$HOME/Documents/budget/2026/依頼書作成.xlsx"
PDF_PATH="$HOME/Documents/budget/2026/提出書類/{YYYYMMDD}_{業者略称}_依頼書.pdf"

# 依頼書以外のシートを一時的に非表示にしてからPDF出力し、保存せずに閉じる
rm -f "${PDF_PATH}"  # 既存PDFが残っているとExcelの上書き保存が失敗することがある

osascript \
  -e 'tell application "Microsoft Excel"' \
  -e 'if not (exists workbook "依頼書作成.xlsx") then' \
  -e "open workbook workbook file name \"${XLSX_PATH}\"" \
  -e 'delay 4' \
  -e 'end if' \
  -e 'set wb to workbook "依頼書作成.xlsx"' \
  -e 'set visible of worksheet "物品一覧" of wb to false' \
  -e 'set visible of worksheet "master" of wb to false' \
  -e 'delay 1' \
  -e 'calculate full rebuild' \
  -e 'delay 3' \
  -e "save workbook as wb filename \"${PDF_PATH}\" file format PDF file format" \
  -e 'delay 2' \
  -e 'set visible of worksheet "物品一覧" of wb to true' \
  -e 'set visible of worksheet "master" of wb to true' \
  -e 'close wb saving no' \
  -e 'end tell'
```

出力後は必ず `mdls -name kMDItemNumberOfPages` とファイルサイズで 1ページ・数十KB以上であることを確認し、Read ツールで内容が依頼書様式になっているか検証する。空白や複数ページの場合は失敗として扱う。

成功したら以下を表示:
```
✅ xlsx に追記しました（OrderID: {N}）
✅ 見積書PDFを 提出書類/{起案日}_{業者略称}_見積書.pdf にアーカイブしました
✅ 依頼書PDFを 提出書類/{起案日}_{業者略称}_依頼書.pdf に保存しました
```

**osascript が失敗した場合**（Excel 未インストール・権限エラー等）は、手動手順を案内する:
1. `依頼書作成.xlsx` を Excel で開く
2. `依頼書`シートへ移動（A2 空 → 最新 OrderID が自動表示）
3. ファイル → PDF として書き出し（印刷範囲のみ）
4. 保存先: `~/Documents/budget/2026/提出書類/{YYYYMMDD}_{業者略称}_依頼書.pdf`

業者略称は業者名から自動生成（例: 「株式会社横山事務器」→「横山事務器」）する。

---

## エラー時の対処

- **ValueError: 必須フィールドが不足**: Step 2 に戻って不足項目を確認
- **FileNotFoundError**: xlsx パスを確認（iCloud 同期が完了しているか確認を促す）
- **業者名 WARN**: master シートへの追記を促す（処理はブロックしない）
- **xlsx が開かれている**: Excel を閉じてから再実行するよう案内
